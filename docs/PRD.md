# Product Requirements Document (PRD): Briefly

## 1. 프로젝트 정의
'Briefly'는 개발자와 직장인이 매일 작성한 투두와 메모를 바탕으로, 주간 업무 리포트를 자동 생성해주는 AI SaaS입니다.

## 2. 핵심 기능 (Features)

### A. 주간 투두 관리
- **날짜 기반:** 사용자는 '이번 주(일~토)' 단위로 업무를 관리한다.
- **CRUD:** 투두 생성, 조회, 수정, 삭제, 완료 처리가 가능하다.
- **메모:** 단순 할 일 제목 외에 상세 메모(트러블 슈팅, 비고)를 기록할 수 있다.

### B. 리포트 자동 생성 & 다운로드 (Report & Export)
- **기본 양식 (Standard Format):**
  - 별도의 템플릿 파일이 없어도 시스템이 제공하는 **'기본 마크다운(Markdown) 양식'**으로 리포트가 생성되어야 한다.
  - **구성:** [기간] -> [요약] -> [진행한 업무] -> [이슈 사항] -> [차주 계획]
- **커스텀 양식 (Advanced - 추후 개발):** 사용자가 업로드한 엑셀/워드 양식에 맞춰 생성하는 기능은 2단계로 미룬다.
- **다운로드:** 생성된 리포트는 **'텍스트 복사'** 또는 **'파일 다운로드(.md or .txt)'**가 가능해야 한다.

## 3. 비즈니스 규칙 (Policies)
- **데이터 범위:** 리포트 생성 시, 선택된 날짜 범위에 포함된 데이터만 참조한다.
- **초기 상태:** 앱 접속 시 무조건 '오늘' 날짜가 선택되어야 한다.
- **예외 처리:** 업로드된 템플릿이 삭제되거나 없는 경우, **다운로드 버튼을 비활성화하는 것이 아니라 '기본 양식(Text Only)'으로 다운로드**되도록 처리한다.
